7. Add proper logging for request and response
8. Implement try-catch for exception handling
9. Use `ApiResponse` format for all responses
10. Follow clean coding standards and naming conventions

Acceptance Criteria:

* Customer can be created successfully
* Customer details can be fetched correctly
* Customer data can be updated

* All APIs return standardized ApiResponse
* Validation errors handled properly
* No unhandled exceptions
* Code is modular and readable
* Logging is implemented correctly

API Details:

Create Customer:

* Method: POST
* Route: /customer/create
* Input: Name, Email
* Output: Customer ID and success message

Get Customer Details:

* Method: GET
* Route: /customer/get
* Input: CustomerId
* Output: Customer details

Update Customer:

* Method: POST
* Route: /customer/update
* Input: CustomerId, Name, Email
* Output: Success message

Delete Customer:

* Method: POST
* Route: /customer/delete
* Input: CustomerId
* Output: Success message
